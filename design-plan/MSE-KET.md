# 🎧 MSE (Mastering Speech & Expressions) 课文精学与语感训练模块设计文档

## 一、模块定位与核心理念

**MSE** 不是简单的“课文阅读器”，而是一个 **“多感官、阶梯式”的语感训练引擎**。 它将传统的“看课文”转化为 **“听辨 ➔ 精读 ➔ 跟读 ➔ 背诵 ➔ 仿写”** 的完整闭环，并与 `VocabMaster`（生词本）底层数据打通，实现“词不离句，句不离篇”。

### 核心学习流 (The MSE Loop)

```text
[ 1. 盲听磨耳 ] ➔ [ 2. 沉浸精读 ] ➔ [ 3. 影子跟读 ] ➔ [ 4. 渐进背诵 ] ➔ [ 5. 句式内化 ]
  (不碰文本)       (扫清障碍)        (肌肉记忆)        (大脑重构)        (输出应用)
```

### 3.2 视觉与排版规范 (MSE 专属)

-   **课文排版 (Typography)**：
    -   **英文**：`1.25rem` (20px)，`Inter`，行高 `2.0`（方便做标记和高亮）。
    -   **中文**：`1rem` (16px)，`Noto Sans SC`，颜色 `var(--text-secondary)`，默认隐藏，悬停或点击句子时淡入。
-   **当前播放句高亮 (Karaoke Effect)**：
    -   当前句子：文字颜色变为 `var(--text-primary)`，背景加上极浅的 `var(--accent-light)` 圆角底色。
    -   非当前句：透明度降至 `0.4`，减少视觉干扰。
-   **词组/句型高亮**：
    -   固定搭配：底部虚线 `border-bottom: 2px dashed var(--warning)`。
    -   核心句型：左侧添加 `3px` 宽的 `var(--accent)` 竖线。

## 四、📄 核心功能与交互细节

### 4.1 阶段一：沉浸精读 (Immersive Reading)

**目标**：扫清听力与阅读障碍，建立声音与意义的连接。

| 交互操作     | 触发方式                | 反馈与效果                                                              |
|--------------|-------------------------|-------------------------------------------------------------------------|
| **点词查义** | 鼠标点击任意英文单词    | 弹出极简气泡：音标 + 核心释义 + 🔊发音 + `[➕ 加入 VocabMaster]` 按钮。 |
| **句级精听** | 点击句子左侧的 `▶` 图标 | 仅播放该句音频，播放完毕自动暂停，等待用户消化。                        |
| **双语切换** | 快捷键 `T` (Translate)  | 译文以 `fade-in` 动画在英文下方展开/折叠。                              |
| **语法拆解** | 选中一段文本，右键/长按 | 弹出“AI 语法解析”卡片（如：主谓宾结构划分、从句标记）。                 |

### 4.2 阶段二：影子跟读 (Shadowing)

**目标**：训练口腔肌肉记忆，纠正语音语调。

```text
┌────────────────────────────────────────────────────────────────┐
│  🎙️ 影子跟读模式 (Shadowing)                                   │
│                                                                │
│  原声波形 (Audio Waveform)                                     │
│  ▂▃▅▇█▇▅▃▂_▂▃▅▇█▇▅▃▂_▂▃▅ (蓝色波形，标记重读与停顿)          │
│                                                                │
│  我的录音 (My Recording)                                       │
│  ▂▃▅█▇▅▃▂_▂▃▅▇█▅▃▂_▂▃▅ (绿色/红色波形，对比原声)              │
│                                                                │
│  Pumas are large, cat-like animals which are found in America. │
│  ^^^^^^      ^^^^^^^^^       ^^^^^^^^^^^                       │
│  (重音高亮)  (连读标记)        (弱读标记)                       │
│                                                                │
│  [▶ 播放原声]   [⏺ 开始录音]   [🔄 重录]   [⏭ 下一句]          │
│                                                                │
│  💡 AI 纠音反馈：                                              │
│  • "cat-like" 连读不够流畅，建议 /t/ 和 /l/ 之间不要停顿。     │
│  • "America" 重音在第二音节 /ˈmer/，你读成了第一音节。         │
└────────────────────────────────────────────────────────────────┘
```

-   **技术实现提示**：前端使用 `Web Audio API` 捕获麦克风并绘制波形；语音评测可接入免费的浏览器 `Web Speech API` 进行基础识别，或接入第三方 API（如阿里云/科大讯飞语音评测）获取精准打分。

### 4.3 阶段三：渐进背诵 (Progressive Recitation)

**目标**：通过“脚手架”理论，逐步撤除提示，实现真正的脱稿背诵。

用户可通过快捷键 `B` 进入背诵模式，系统提供 **4 个阶梯难度**（按 `Tab` 键切换）：

| 阶梯                    | 视觉呈现 (以 "Pumas are large animals" 为例)                | 交互逻辑                                     |
|-------------------------|-------------------------------------------------------------|----------------------------------------------|
| **Level 1: 核心词挖空** | `Pumas are [ ___ ] animals which are [ ___ ] in America.`   | 点击空白处或按空格，单词如打字机般浮现。     |
| **Level 2: 首字母提示** | `P___ a___ l___, c___-l___ a___ w___ a___ f___ i___ A___.`  | 考验拼写与句子结构记忆，点击显示全词。       |
| **Level 3: 逻辑骨架**   | `[ 主语 ] are [ 表语 ] which are [ 过去分词 ] in [ 地点 ].` | 仅保留语法骨架，要求用户用原词或同义词还原。 |
| **Level 4: 全盲回忆**   | `[ 点击此处，在脑海中完整复述本句，然后按回车验证 ]`        | 纯靠大脑重构，验证时显示原文对比。           |

### 4.4 阶段四：句式工坊 (Pattern Workshop)

**目标**：将课文中的“死知识”转化为写作口语的“活武器”。

```text
┌────────────────────────────────────────────────────────────────┐
│  🛠️ 句式工坊: 强调句型 (It is/was ... that ...)                │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  📖 课文原句：                                                 │
│  It was not until he arrived that we started the meeting.      │
│                                                                │
│  🧩 句式抽象：                                                 │
│  It + be + [ 强调部分 ] + that + [ 剩余部分 ]                  │
│                                                                │
│  ✍️ 替换练习 (Drag & Drop 或 下拉选择)：                       │
│  It was [ because of the rain ▼ ] that [ the match was canceled ]│
│                                                                │
│  🚀 自由造句 (你的创作)：                                      │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │ It was                                                  │  │
│  └──────────────────────────────────────────────────────────┘  │
│                                                                │
│  [ ✨ AI 润色与纠错 ]             [ 💾 存入我的句式本 ]          │
└────────────────────────────────────────────────────────────────┘
```

## 五、💾 数据模型设计 (IndexedDB)

MSE 的数据结构以 **Sentence (句子)** 为核心颗粒度，而非整篇课文。

```javascript
// ===== 1. 课文元数据 (Article) =====
const Article = {
  id: "art_nce3_01",
  title: "A Puma at Large",
  source: "新概念英语第三册",
  level: "B2",               // CEFR 难度等级
  tags: ["叙事", "动物", "经典"],
  audioUrl: "/assets/audio/nce3_01.mp3",
  totalTime: 185,            // 总时长(秒)
  coverImage: "...",
  createdAt: "2026-06-01"
};

// ===== 2. 句子数据 (Sentence) - 核心！ =====
const Sentence = {
  id: "sent_nce3_01_001",
  articleId: "art_nce3_01",
  index: 1,                  // 第几句
  
  // 文本与翻译
  textEn: "Pumas are large, cat-like animals which are found in America.",
  textCn: "美洲狮是体形似猫的大型动物，产于美洲。",
  
  // 音频时间戳 (用于同步高亮)
  audioStart: 0.0,
  audioEnd: 4.5,
  
  // 语言学标记 (用于高亮和背诵)
  chunks: [                  // 意群划分 (用于跟读停顿)
    { text: "Pumas are", type: "subject" },
    { text: "large, cat-like animals", type: "object" },
    { text: "which are found in America.", type: "clause" }
  ],
  keyVocab: ["puma", "cat-like"], // 关联 VocabMaster 的单词ID
  patterns: ["passive_voice", "relative_clause"], // 包含的句式ID
  
  // 用户学习状态
  status: {
    listenCount: 3,          // 精听次数
    shadowScore: 85,         // 跟读最高分
    reciteLevel: 2,          // 背诵达到的最高阶梯 (0-4)
    isBookmarked: false      // 是否收藏为金句
  }
};

// ===== 3. 句式模型 (Pattern) =====
const Pattern = {
  id: "pat_emphasis_01",
  name: "强调句型 (It is/was... that)",
  formula: "It + be + {focus} + that + {rest}",
  explanation: "用于强调句子的主语、宾语或状语...",
  examples: [
    { en: "It was yesterday that he arrived.", cn: "他是昨天到的。" }
  ]
};
```

## 六、📥 课文导入与解析引擎

为了支持用户自定义导入，MSE 需要一个强大的前端解析器。

### 6.1 支持的导入格式

1.  **LRC 歌词文件**：完美契合“带时间戳的文本”，直接解析为 Sentence 数组。
2.  **SRT 字幕文件**：同上，自动提取时间轴和中英文本。
3.  **Markdown + 音频**：用户导入 `.md` 课文文件和 `.mp3`，系统通过自然语言处理（NLP）库（如 `compromise`）自动进行**分句**和**时间轴预估**。
4.  **TXT 纯文本 + AI 对齐**：输入纯文本和音频，调用浏览器端或后端 API 进行 Forced Alignment（强制对齐，生成时间戳）。

### 6.2 导入界面交互

```text
┌─────────────────────────────────────────────────────────────┐
│  📥 导入新课文                                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  [ 📄 拖拽文件到此处，或点击上传 ]                            │
│  支持: .mp3/.wav (音频), .lrc/.srt (字幕), .md/.txt (文本)   │
│                                                             │
│  ─── 智能解析选项 ───────────────────────────────            │
│  ☑️ 自动分句 (基于标点符号)                                  │
│  ☑️ 自动提取生词 (对比四六级词表，高亮超纲词)                 │
│  ☑️ 生成双语对照 (调用免费翻译API)                           │
│                                                             │
│  [ 取消 ]                                   [ 🚀 开始解析 ] │
└─────────────────────────────────────────────────────────────┘
```

## 八、🔗 与 VocabMaster 的深度联动

MSE 不是孤岛，它是 VocabMaster 的**最佳内容供应商**。

1.  **语境化导入**：在 MSE 中点击单词加入生词本时，**自动将当前课文句子作为“例句”带入 VocabMaster**。
    -   *VocabMaster 闪卡显示*：单词 `puma` ➔ 翻转 ➔ 释义 + 例句 "Pumas are large, cat-like animals..." + 🔊播放课文原声片段。
2.  **课文反查**：在 VocabMaster 复习时，如果单词来源于 MSE 课文，卡片底部会出现 `[📖 查看课文上下文]` 按钮，点击直接跳转回 MSE 的对应句子。
3.  **熟词标记**：如果在 VocabMaster 中某词已标记为“掌握(5级)”，在 MSE 课文中该词会自动降低透明度，让用户聚焦于生词。

## 九、✨ 提升高级感的微交互细节 (Micro-interactions)

| 交互场景         | 动效与视觉设计                                                                                               |
|------------------|--------------------------------------------------------------------------------------------------------------|
| **音频加载**     | 播放按钮呈现 `breathing` 呼吸动画，波形图以骨架屏(Skeleton)微光扫过加载。                                    |
| **句子高亮切换** | 当前句背景色使用 `transition: all 0.3s ease` 平滑过渡，非当前句 `opacity` 渐变。                             |
| **跟读打分**     | 录音结束后，分数数字使用 `countUp` 滚动动画，≥80分爆发出微弱的绿色粒子(Particle)效果。                       |
| **背诵填空**     | 单词出现时，不使用生硬的 `display: block`，而是使用 `letter-spacing` 从紧缩到正常的打字机效果。              |
| **加入生词本**   | 点击 `➕` 后，单词化作一个微小的光点，沿着贝塞尔曲线飞入右上角的 VocabMaster 图标，并伴随清脆的 `pop` 音效。 |

## 十一、技术栈补充建议 (针对 MSE 模块)

-   **音频处理**：`Howler.js` (处理音频播放、循环、时间轴更稳定) 或 原生 `HTML5 Audio API` + `AudioContext` (用于波形分析)。
-   **波形可视化**：`Wavesurfer.js` (业界标准的音频波形库，支持Regions标记句子时间轴)。
-   **文本解析**：`compromise` (轻量级前端 NLP 库，用于英文分词、词性标注、提取动词短语)。
-   **动画库**：`GSAP` (处理复杂的贝塞尔曲线飞入动画和打字机效果，性能极佳)。

>   **开发者寄语**：MSE 模块的开发难点在于 **“音频时间戳与文本的精准对齐”** 以及 **“跟读录音的延迟处理”**。建议在 Phase 1 阶段，优先支持用户导入带有时间轴的 `.lrc` 或 `.srt` 文件，将“自动对齐”这个深水区留到 Phase 5 结合 AI 解决，以保证前期开发的顺畅与产品的快速可用。
