# 🎓 ClassFlow (教务与学情追踪引擎) 模块设计文档

## 一、模块定位与核心理念

**ClassFlow** 是专为独立教师/小班教学设计的轻量级教务管理中枢。 它摒弃了传统教务系统臃肿复杂的表单，以 **“时间轴”** 和 **“学生档案”** 为双核心，将签到、课程记录、作业追踪无缝融合，让教师将精力回归教学本身。

### 核心设计原则

1.  **极简录入 (Frictionless Entry)**：签到和布置作业的操作必须在 3 次点击内完成。
2.  **状态可视化 (Visual Status)**：用颜色和图标直观呈现“出勤状态”与“作业进度”，拒绝密集的文字表格。
3.  **数据沉淀 (Data Accumulation)**：每一次签到和作业提交，自动汇聚成每个学生的“个人学情画像”。

## 五、📁 作业文件保存与存储方案 (关键技术落地)

由于涉及音频、图片、PDF 等文件，纯前端 LocalStorage 无法满足需求。推荐以下两种落地架构：

### 方案 A：云原生 BaaS 架构（🌟 强烈推荐，适合独立开发者）

利用 **Supabase** 或 **Firebase** 提供的一站式后端服务，免运维，自带文件存储。

-   **数据库**：Supabase PostgreSQL (存储上述数据模型)。
-   **文件存储 (Storage)**：Supabase Storage / Firebase Storage。
    -   **上传流程**：前端选择文件 ➔ 生成唯一 UUID 文件名 ➔ 调用 SDK 上传至 Storage Bucket ➔ 获取 Public URL ➔ 将 URL 存入 `Submission` 表的 `files` 数组中。
    -   **权限控制 (RLS)**：配置 Row Level Security，确保学生只能读取/上传自己的作业，教师可读写所有。
-   **成本**：Supabase 免费额度包含 1GB 存储和 2GB 带宽，对于初期小班教学完全足够。

### 方案 B：纯前端 + 对象存储 (适合已有云服务账号)

-   **前端**：Vue/React。
-   **存储**：阿里云 OSS / 腾讯云 COS / Cloudflare R2。
-   **流程**：前端通过 STS (Security Token Service) 获取临时凭证，直接将文件直传至 OSS，不经过自己的服务器，节省带宽成本。

### 文件预览支持

-   **图片**：直接 `<img>` 标签渲染。
-   **音频**：使用 `<audio controls>` 或集成 `Wavesurfer.js` 绘制波形（与 MSE 模块复用组件）。
-   **PDF**：使用 `pdf.js` 进行内嵌预览。
-   **Word/PPT**：调用微软 Office Web Viewer API (`https://view.officeapps.live.com/op/view.aspx?src=文件URL`) 进行免登录预览。

## 六、✨ 提升高级感的微交互与细节

| 场景                     | 交互与动效设计                                                                                                     |
|--------------------------|--------------------------------------------------------------------------------------------------------------------|
| **签到状态切换**         | 点击卡片时，边框颜色以 `0.2s` 平滑过渡（如灰变绿），伴随轻微的 `scale(0.98)` 按压反馈和清脆的 `tick` 音效。        |
| **作业拖拽上传**         | 拖入文件时，上传区域背景变为 `var(--accent-light)`，边框变为虚线并呈现 `breathing` 呼吸动画。                      |
| **逾期预警**             | 逾期未交的作业卡片，右上角的时钟图标呈现红色，并带有微弱的 `shake` (摇晃) 动画，吸引教师注意。                     |
| **成绩发布**             | 点击“提交并通知”后，按钮变为 `✅ 已发送`，并伴随一个纸飞机飞出的微动效（寓意通知已发送给学生）。                   |
| **空状态 (Empty State)** | 当没有待批改作业时，显示一张精致的 SVG 插画（如一杯咖啡），配文“太棒了！所有作业都已批改完毕，享受休息时间吧 ☕️”。 |

## 七、📱 移动端适配策略 (Teacher on the go)

教师经常在教室走动，移动端体验至关重要：

1.  **签到台移动端化**：网格布局变为**垂直列表**，左侧头像+名字，右侧提供巨大的滑动切换按钮（Swipe to change status）。
2.  **快捷扫码签到**：移动端增加“生成签到二维码”功能，学生扫码后自动标记为“出勤”（需配合简单的后端鉴权）。
3.  **语音转文字评语**：在移动端批改作业时，提供“麦克风”图标，调用系统语音识别，教师直接口述评语，自动转为文字，极大提升批改效率。

## 九、技术栈推荐补充

-   **后端与数据库**：**Supabase** (开源 Firebase 替代品，PostgreSQL 极其强大，自带 Auth 和 Storage，对前端开发者极度友好)。
-   **前端框架**：**Vue 3 (Composition API)** 或 **React** + **Tailwind CSS** (快速构建高级感 UI)。
-   **图表库**：**Apache ECharts** (功能全面) 或 **Chart.js** (轻量美观，推荐后者以匹配极简设计)。
-   **日期处理**：**Day.js** (轻量级，处理课程时间计算、逾期判断)。
-   **文件处理**：**FilePond** (极其优雅的文件上传 UI 组件库，自带拖拽、预览、进度条)。

>   **开发者寄语**：ClassFlow 的核心价值在于 **“减负”**。在开发时，请时刻问自己：“这个操作能不能再少点一次鼠标？” 将签到和作业布置做到极致的顺滑，这个模块就会成为您日常教学中不可或缺的生产力工具。
